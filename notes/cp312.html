Big Oh Notation

If f(n) = Og(n); f(n) is bounded above by g(n) OR f(n) is not faster than g(n).

Definition

f(n) in Og(n) iff f(n) <= cg(n) where c is a constant for all n >= n0.

Example

f(n) = 10n
g(n) = n^2
 
n^2 = 10n
n^2 - 10n = 0
n(n - 10) = 0
n = 0 or n = 10
 
n0 = 10, c = 1
 
10n in O(n^2)
 
10n is always going to be bounded by n^2.

Example

f(n) = 5n + 20
g(n) = n
 
5n + 20 = n
4n = -20
n = -5
 
c = 100, n0 = -5
 
5n + 20 in O(n) -> 5n + 20 will not grow faster than n.

Big Omega Notation

Definition

f(n) in Ωg(n) iff f(n) >= cg(n) where c is a constant for all n >= n0. Ex. cg(n) is a lower bounded for f(n).

Big Theta Notation

Definition

f(n) in ϑg(n) iff f(n) >= c1g(n) and f(n) <= c2g(n) where c1 and c2 are constants for all n >= n0. Thus, f(n) and g(n) are the same growth rate.

Example How to compare the growth rate of two given functions?

f(n) = 3n^2 + n + 5 and g(n) = 5n^3 + 5n + 6

We are only interested when the input size is large; where large is infinity.

lim(n -> inf)  f(n) / g(n) = lim(n -> inf) (3n^2 + n + 5) / (5n^3 + 5n + 6)

There will be three cases: 0 or constant > 0 or inf.

                                0 => f(n) in 0(g(n))
lim(n -> inf) f(n) / g(n) =     constant > 0 => f(n) in ϑ(g(n))
                                inf => f(n) in Ω(g(n))

Example

f(n) = 10n and g(n) = n^2.

lim(n -> inf) = lim(n -> inf) = 10n / n^2
              = lim(n -> inf) 10/n
              = 0

Thus 10n in O(n^2).

Example

n(n+1)/2 and n^2.

lim(n -> inf) n(n+1)/2 / n^2 = lim(n -> inf) n^2 + n / 2n^2
                             = lim(n -> inf) (n^2/2n^2 + n/2n^2)
                             = lim(n -> inf) (1/2 + 1/2n)
                             = 1/2

Thus n(n+1)/2 in ϑ(g(n)).

Example

log2(n) and n. Use L'Hopital's rule.

lim(n -> inf) log(n) / n = lim(n -> inf) ð log(n) / ð n
                         = lim(n -> inf) 1/n*ln2
                         = 0

Thus, f(n) in O(g(n)). f(n) is always slower than g(n).

Observations

  • f(n) in O(f(n))
  • f(n) in O(g(n)) => g(n) in Ω(f(n))
  • If f(n) in O(g(n)) and g(n) in O(h(n)) then f(n) in O(h(n))
  • If f1(n) in O(g1(n)) and f2(n) in O(g2(n)) => f1(n) + f2(n) in O(max { g1(n), g2(n) })

Analyze the Time Efficiency of Non-Recursive Algorithms

  1. Identify the input size
  2. Identify basic operation
  3. Determine worst, average and best case for input size n
    • Mainly just look at the worst case, but average can give you an idea ofhow long the algorithm will take
  4. Set up a sum for the number of times the basic operations is executed
  5. Simplify the sum
  6. Determine time efficiency class

        = 2^k-1 (2M(n/2^k) + (n/2^k-1 - 1)) + (n - 2^k-2) + ... + (n - 2^1) + (n - 2^0)

    M(1) = 0

    Worst case: T(n) in O(n^2)
    Best case: T(n) in O(n log(n))

Let T(n) be an eventually non-decreasing function that satisfies the recurrence.

T(n) = a*T(n/b) + f(n)
T(1) = 1
 
where a >= 1, b >= 2, c > 0
 
         ϑ(n^d)             if a < b^d

Brute Force Algorithms

A straight forward approach to solving a problem, usually directly based on the problem statement and definitions of the concept involved. Use of force of the computer instead of a clever approach.

Example Find a^n

a = a * a * ... * a       n times
 
power = a
for i = 1 to n - 1
  power *= a

Strengths

  • Applicable to a wide variety of problems
  • For some problems, brute force algorithms yield reasonable algorithms
  • A more efficient algorithm may not be justified if the brute force is already fast enough
  • Even if the approach is inefficient, it can find the solution for small input size
  • Serve as comparison algorithm to other algorithms

Weaknesses

  • Rarely yields efficient algorithms
  • Can be very slower
  • May not work if input size is large

Convex Hull Problem

The convex hull, or convex set is the smallest set, of a set X, containing X. Given the values of points in a 2d space, sub them into the equation to find the equation of the line (ex. ax + by + c = 0). In order to determine if the lines lies in the convex hull, check every point and sub into equation.

Time Efficiency

For each point it can form (n-1) point pairs. The total number of pairs is n(n-1) / 2. Check (n-2) points for each pairs. The total number of operations: n(n-1) / 2 * (n-2). Thus, T(n) in O(n^3).

Chapter 11 - Limitations of Algorithms

  • Difficult to find the lower bounds of some problem P
  • Much easier to find the lower bound for some problem T of P
    • But the lower bound of T is not the lower bound for P
    • There may be other solutions T1, T2, etc of different time efficiency brackets

Problem Reduction

  • We want to find a lower bound for problem P
  • Get a problem Q with a known lower bound
  • Take an instance of Q and transform it into P: Q => P
  • If the transformation can be done in polynomial time:
    • Any algorithm that can solve P can solve Q
    • P is as hard as Q
    • Lower bound of Q is the lower bound of P

Definition

An algorithm that can solve a problem in polynomial time for its worst case belongs in O(p(n)) where p(n) is the polynomial of the problems input size.

  • Input size is the order of the polynomial

Definition

A problem is tractable if it is in O(p(n)) otherwise it is non-tractable

  • ex. It can be solved in polynomial time IF tractable

Decision Problem [yes/no problems]

  • Any optimization problem can be transformed into a decision problem
  • ex. A minimum path problem can be transformed into a decision problem: is this the optimal path?

Definition

Class P is a class q decision problem that can be solved in polynomial time by a deterministic algorithm.

Definition

A non-deterministic algorithm is a 2 stage procedure that takes its input, an instance I of a decision problem and does the following:

  1. Non-deterministic stage: Generate a solution to I (by guessing)
  2. Use deterministic (polynomial time) algorithm to verify the solution generated in the non-deterministic shape

Definition

Class NP is the class q decision problem that can be solved by a non-deterministic polynomial algorithm (solution can be determined to be correct or incorrect in polynomial time).

  • P: can be solved in polynomial time
  • NP: cannot be solved in polynomial time

P is a subset of Q

  • Is P = NP or P != NP?

Branch and Bound Algorithms

Knapsack Problem

  • Only works for optimization problems
  • Ex. given a knapsack of a fixed size, and a number of items; maximize the number of items that go into the backpack
    • Calculate the value/pound ratio
    • At each step, the upper bound is the sum of the item's value currently in the bag, plus the remaining weight times, the highest value/pound of an item still being considered
    • Starting with the highest value/pound item, consider the UB of including the item and not including it, (including on left, discluding on right)
    • Expand the node with the highest UB with the inclusion of the a node for including the next best value/pound and not including it
    • Keep expanding the nodes with the highest upper bounds, including the next value/pound each time
    • Stop until you get solution that is higher than the UB of the remaining leafs

Assignment Problem

  • Goal is to minimize assignment cost
  • Create a start node for the first job
    • Lower bound is sum of lowest row value for each row, as long as the rows value isn’t in the same column as the jobs already selected on this path
  • Initially expand start node with all possible assignments for person A
  • Expand the assignment with the smallest lower bound, including all assignments for person B
  • Keep doing this until a solution is found with a cost less than or equal to all other node’s lower bounds
