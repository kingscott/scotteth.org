Assembler Directives

  • Instructions to the assembler while your program is being assembled
  • They do NOT generate machine code

Examples

  • ORIGIN: Specifies a memory location where the next piece or data is located (don't start below $400)
    • Addresses are always in hex
  • END: Last line of code in program
    • Stops assembly process

  • EQUATE: Equates a symbol with a number

    • Usage:
    WEEKS   EQU   52
    DAYS    EQU   WEEKS*7
    ...

  • Define Constant: Allocate and initialize a memory location

    • Usage: DC.B, DC.W, DC.L
  • Define Storage: Allocate a specified amount of memory, but does not initialize it
    • Usage: DC.B, DC.W, DC.L

Integer Representation

Unsigned numbers are assumed to be positive and no bits are used to represent the sign. Signed numbers follow this:

  1. Positive and negative numbers
  2. Sign test - test whether negative or positive
  3. Zero test - test whether the number is 0
  4. Easy implementation of arithmetic. In order of important: addition, subtraction, multiplication, division

Secrets

  • Signed magnitude uses the first bit to determine sign; 1 for negative, 0 for positive
  • Diminished radix complement in base 2 is 1's complement (9's complement in base 10)
  • Radix complement in base 2 is 2's complement (10's complement in base 10)

Note: Positive numbers are ALWAYS stored the same, whether unsigned, signed, 1's or 2's complement.

+6 -6
unsigned 00000110 ---
signed-magnitude 00000110 10000110
1's complement 00000110 11111001
2's complement 00000110 11111010

Character Representation

Addressing Modes

  • Defines how data is accessed
  • Orthogonality property: every addressing mode would work with every instruction
  • Effective address, <ea> or EA; indicates how the address of the operand is calculated
  • All addressing modes, except immediate have an EA
ADD   Dn,<ea>      ; OR
ADD   <ea>,Dn

Basic Addressing Modes

1. Immediate

  • # indicates that the item that follows is immediate data (Ex. #4, #The, #%1001)
  • <ea> = none
MOVE.B    #25,D0     ; where D0 is 00 00 00 19
                     ; #25 = $19
 
ADDI.B    #1,D2

2. Register Direct (Dn or An)

  • Operand is a general register
  • <ea> = Rn
MOVE.B    #25,D0      ; direct
 
ADDI.B    #1,D2       ; immediate

3. Memory Direct

  • Operand is a variable +- constant expression (lazily evaluated)
  • The address of the operand is part of the instruction
  • <ea> = address
        ...
        MOVE.B  D0,num1     ; memory direct
 
num1    DC.B    $05

Also:

MOVE.B    25,D0       ; will move whatever is in memory location 25 to D0
  • (xxx).W or Abs.W (stored as 16 bits) [absolute short]
    • Ranges from $0 to 7FFF and $8000 to $FFFF
  • (xxx).L or Abs.L (stored as 32 bits) [absolute long]
  • Abs. short generates the more compact machine code

Status Register

  • Status = system byte (supervisor only) + user byte -> system status + Condition code register

Note: Order of bits usually doesn't matter.

T (trace)

  • 1 in trace mode, 0 otherwise
    • Single step instruction trace

S (supervisor)

  • 1 in supervisor mode, 0 is user mode

Example

Looking at: STOP #$2700, actually flips S bit to 1

N (negative)

  • 1 is the most significant bit of an operand or result is 1

Z (zero)

  • 1 is all bits in the operand or result are 0

V (overflow)

  • 1 if an out of range condition, overflow (in signed arithmetic)

C (carry)

  • 1 if a carry is generated out of the most significant bit during arithmetic operation (overflow in unsigned arithmetic)

X (extend)

  • Used in multiple precision arithmetic

Using Condition Codes

  • The machine does 2's complement and generates ALL flags
  • The machine doesn't know if it is unsigned or signed
  • You pick the flags appropriate to your operation

Loops

  • Count usually goes down to zero (easy to compute)

Steps for assembly loop:

  1. Initialize count register (ideal m8)
    • Decrement count
    • {body of loop}
    • BNE loop
  2. Initialize count register (check against 0 - skip loop)
    • BEQ skip
    • {body of loop}
    • Decrement count
    • BNE loop
    • skip {rest of the code}

Subroutines

  • Procedures, or functions in high level languges are modeled on subroutines
  • Assembly code is usually very modular
  • JSR, jump to subroutine, used to call subroutine
  • RTS, return from subroutine

Example

        JSR     PRT_S           ; jump to subroutine PRT_S
        ...
        LEA     msg2,A2         ; point to string

  1. Jump to subroutine: JSR <ea>
    • Unlimited addressing range
    • Stores status register address in machine code
  2. Branch to subroutine: BSR <ea>
    • Smaller machine code, limited addressing range
    • Stores the displacement from the current instruction

Key Registers

PC: Program Counter (aka IP: instruction pointer)

  • Contains the address of the next instruction to be executed
  • To keep track of where we are in the program
    • Sends the PC to memory
    • While memory is retrieving the instruction, PC is instructed to point to new instruction
    • Memory returns instruction
    • CPU executes instruction

A7: Stack Pointer (SP)

  • Not used as a regular status register, used as system stack pointer
  • In supervisor mode: SSP (supervisor stack pointer)
    • Used USP to access user stack pointer
  • If you are in user mode, USP
    • SSP not visible in this mode
  • SSP and USP are physically separate registers

Register Transfer Language with JSR/RTS

JSR:

  1. Save program counter on stack
  2. Execution continues at specified address
(SP) <- (SP) - 4
((SP)) <- (PC)
(PC) <- address

RTS:

  • RTS - get PC from stack and return
(PC) <- ((SP))
(SP) <- (SP) + 4
  • Return and restore (RTR) condition codes
    • Get conditions from stack
    • Get PC from stack and return
(CCR) <- ((SP))      ; ((...)) store
(SP) <- (SP) + 2
(PC) <- ((SP))
(SP) <- (SP) + 4
  • Stacks grows down in memory
  • SP points to the last filled location in the stack

Defining the System Stack

  1. If there is any operating system, the operating system will initialize the SP
  2. If there is a monitor program, check the SSP/USP registers
  3. If there is no monitor program, if it doesn't initialize SSP/USP, it is the programmer's responsibility to initialize the USP/SSP in the 1st line of the program:
    • If stack is at $10000, use:
LEA     $10000,A7
LEA     $10000,SP

Saving/Restoring Context

  • Programmer's responsibility to save registers when entering the status register and restore registers when exiting the SR
    • Save registers that the SR will modify
    • Don't save registers that are used to pass values
  • Save/restore registers with move multiple
  • MOVEM.L D0-D3/A2-A5,-(SP)

Loop Structures (DBcc)

  • A decrement branch instruction handles two concurrent conditions
    1. One conditional test
    2. One loop count

DBcc Dn,<label>

  1. If the condition being tested is satisfied, control passes to the instruction after the DBcc
  2. If the condition is not satisfied, then
    • The lower 16 bits of Dn is decremented by one
    • If the result is -1: control passes instruction following the DBcc
  3. Otherwise control is transferred to label

Decimal Arithmetic

  • Numbers are stored as a series of base 10 digits
    • BCD* or decimal arithmetic

There are two formats for BCD:

  • Unpacked, where 8 bits = 1 decimal digit
    • Ex. 198510 = %0000 0001 0000 1001 0000 1000 0000 0101
  • Packed, where 8 bits = 2 decimal digits
    • Ex. 198510: %0001 1001 1000 0101

Why BCD?

  • I/O systems are usually ASCII and keypad systems are BCD
  • Arithmetic operations:
    • Work on one byte at a time
    • Assume packed BCD --> 8 bits = 2 decimal bytes
    • Unpacked BCD is only used in transition from ASCII to BCD --> 8 bits = 1 decimal digit in low nibble

Operations with BCD

  • Start at the least significant byte work towards the most significant byte
  • Carry handled using X

Flags for BCD

  • Clear X (extended carry) before first BCD operation so that carry is handled correctly
  • N and V are undefined for decimal arithmetic
    • Sign is explicitly handled during the operation
  • C and X is set if decimal carry/borrow occurs
    • Can be used to construct result if overflow
  • Z is changed if the result is nonzero; otherwise unchanged
    • For addition, set Z to 1 before first BCD operation to obtain the correct Z (optional)
    • No requirement to initialize Z if it is not important to the problem

Addressing Modes (on Address Registers)

  1. Immediate: #<data>
    • Data is part of the instruction
  2. Register direct: Dn or An
    • Data is in the register
    • The address of the data is the register number
  3. Memory direct: (xxx).W or (xxx).L
    • Data is in the memory location
    • The address of the data is the address in the memory location
  4. Address register indirect (An)
  5. Postincrement: (An+)
    • The address of the data is in the address register
  6. Predecrement: -(An)
    • The address of the data is in the contents of the address register minus the decrement
  7. Address register indirect with displacement: d(An)
    • Operand is written as d(An), where d = 16 bit displacement
    • The address of the data is the sum of the address register and displacement
  8. Address register indirect with indexing: d(An, Rn)
    • Operand is written as d(An, Rm{.W|.L}); where d = 8 bit displacement and Rm is an index register (W or L)
    • The address of the data is the sum of the address register, the index register and the displacement
  9. Program counter relative w/ displacement: d(PC)
    • Operand is written as d(PC) where d = 16 bit displacement
    • The address of the data is the sum of the program counter and the displacement
  10. Program counter relative w/ indexing: d(PC, Rn)
    • Operand is written as d(PC, Ri{.W|.L}) where d = 8 bit displacement
    • The address of the data is the sum of the PC, the index register and the displacement

Position Independent

  • Code can be loaded and executed from any starting address in memory after it has been assembled; address assigned at run time
    • Static --> assembly, ex. shared library, ROM based program
    • Dynamic --> can be moved after it has started execution, ex. virtual system memory
  • No absolute addresses except for hardware device addresses
  • PC-relative addressing is used wherever possible for branches within modules
  • Where PC-relative addressing cannot be used, indirect addressing through a linkage table is used for accesses to global variables, for inter-module procedure calls, for branches and literal accesses
    • The dynamic loader fills in procedure and data linkage tables with the absolute virtual addresses of the routines and data in a shared library at run time (known as binding)

Relocatable vs Non-Relocatable Code

  • Relocatable code

    • Code can be moved to a different starting address but requires reassembly
    • Code contains symbols but no explicit addresses (except hardware)
    • Addresses assigned at link time

  • Non-relocatable code

    • Code requires reprogramming to be moved to a different memory location
    • Code contains actual addresses
    • Addresses assigned during programming
