Introduction

How Regular Languages Work

C-like languages:

            compile                                     compile            loader
HLL program   ->    intermediate code (pseudo-assembly)   ->   object code  |->|  machine code

            compile              VM loader             VM compiler
HLL program   ->    portable code  |->|   VM memory image  ->  machine code

General Information On Assembly Language

  • Microprocessors and assembly languages are modularized -> one compiler for one mCPU and language

Structure Of Syntax

label opcode operand(s) comment

  • Label: Represents an address
  • Opcode: Operation to be performed; + specific length: B, W, L
  • Operand: 1st operand is the source, 2nd operand is the destination

Assembler Directives

  • Instructions to the assembler while your program is being assembled
  • They do NOT generate machine code

Examples

  • ORIGIN: Specifies a memory location where the next piece or data is located (don't start below $400)
    • Addresses are always in hex
  • END: Last line of code in program
    • Stops assembly process

  • EQUATE: Equates a symbol with a number

    • Usage:
    WEEKS   EQU   52
    DAYS    EQU   WEEKS*7
    ...

  • Define Constant: Allocate and initialize a memory location

    • Usage: DC.B, DC.W, DC.L
  • Define Storage: Allocate a specified amount of memory, but does not initialize it
    • Usage: DC.B, DC.W, DC.L

Example - Program to add two numbers [and store result]

Inputs: NUM1, NUM2 (byte length) Output: RESULT (byte length) Algorithm:

D0 <- NUM1
D1 <- NUM2
D1 <- D0 + D1
RESULT <- D1
        ORG       $1000
 
START   MOVE.B    num1,D0      ; get first number
        MOVE.B    num2,D1      ; get second value
        ADD.B     D0,D1        ; actually adds the numbers
        MOVE.B    D1,result    ; store result
 
        STOP      #$2700
 
num1    DC.B      10
num2    DC.B      27
result  DS.B      1
 
        END       START

  • The size of a byte stays the same across all assembly languages
  • The size of a word is dependent on the natural systems length (architecture)

Integer Representation

Unsigned numbers are assumed to be positive and no bits are used to represent the sign. Signed numbers follow this:

  1. Positive and negative numbers
  2. Sign test - test whether negative or positive
  3. Zero test - test whether the number is 0
  4. Easy implementation of arithmetic. In order of important: addition, subtraction, multiplication, division

Secrets

  • Signed magnitude uses the first bit to determine sign; 1 for negative, 0 for positive
  • Diminished radix complement in base 2 is 1's complement (9's complement in base 10)
  • Radix complement in base 2 is 2's complement (10's complement in base 10)

Note: Positive numbers are ALWAYS stored the same, whether unsigned, signed, 1's or 2's complement.

+6 -6
unsigned 00000110 ---
signed-magnitude 00000110 10000110
1's complement 00000110 11111001
2's complement 00000110 11111010

Character Representation

Characters are usually represented using ASCII. In 68000, characters are represented using single quotes; '123' = 31 32 33 / character = hex.

M68000 Programming Model

  • The 'programming model' is the computer architecture that the programmer can access
  • If you are access a register at less than the full length, the operation treats the rest of the register as if it does not exist

Data registers: [D0-D7]

  • 0 to 7 is a byte
  • 0 to 15 is a word
  • 0 to 31 is a longword

Address registers: [A0-A7]

  • Cannot access address register at a byte - you can only move/access words

Stack Pointer

Note: 68000 (EASy68K) runs in supervisor mode by default

  • 2 stack pointers since 68000 can run in user mode (applications) or supervisor mode (operating system)

    • SSP: Supervisor Stack Pointer

Example Copy 16 bits of data from one memory location to another

  • Algorithm: num2 <- num1

  • Move to an intermediary register (most ideal)

START   ORG     $1000
        MOVE.W  num1,D0   ; get value
        MOVE.W  D0,num2   ; and store
        STOP    #$2700
 
num1    DC.W    $2E56
num2    DS.W    1
 
        END     $1000
  1. Move directly to num2
        ...
        MOVE.W  num1,num2
        ...
  1. Not ideal - but can be done
        ...
        MOVE.B  num1,num2       ; would only put the first byte of num1 into num2
        MOVE.B  num1+1,num2+1   ; this is needed to move the second byte in there too
  1. Using an improper length suffix
        ...
        MOVE.L  num1,num2       ; we are going to get all of num1 (word) and whatever word of data is in num2: MUMBO JUMBO
        ...                     ; this is alright for the way the data is setup now, but ...
 
num2    DS.W    1               ; if we had it stored like this, it would overwrite num1
num1    DC.W    $2E56

Example Addition of two 16 bit numbers

  • Algorithm:
    • D0 <- num1
    • D1 <- num2
    • D1 <- D0+D1
    • sum <- D1

1. Standard way

        ...
        MOVE.W  num1,D0     ; get first value
        MOVE.W  num2,D1     ; and the second
        ADD.W   D0,D1       ; add values
        MOVE.W  D1,sum      ; store output
        ...
 
num1    DC.W    $1075
num2    DC.W    $0625
sum     DS.W    1

2. Alternative: minus one operation (preferred) [4 memory accesses]

        ...
        MOVE.W  num1,D0     ; grab first value
        ADD.W   num2,D0     ; add second value directly to register
        MOVE.W  D0,sum      ; store
        ...

3. Using an one intermediary - but uses result memory automatically [5 memory accesses]

        ...
        MOVE.W  num1,sum    ; put first in result memory
        MOVE.W  num2,D0     ; put in intermediary register
        ADD.W   D0,sum      ; read from memory, add, write to memory
        ...

Example Takes 1's complement of 16 bit value stored in memory

  • Input: $7F3E -> 0111 1111 0011 1110
  • Ouput: $7F3E => $80C1 == 1000 0000 1100 0001

1. Standard

        ...
        MOVE.W  num,D0      ; grab value
        NOT.W   D0          ; 1's complement
        MOVE.W  D0,num      ; rewrite old num and store
        ...
 
num     DC.W    $7F3E

2. Take 1's complement directly in memory

        ...
        NOT.W  num          ; do it directly
        ...

3. Subtraction method

        ...
        MOVE.W  #$FFFF,D0   ; move direct value to register
        MOVE.W  num,D1      ; move num
        SUB.W   D1,D0       ; takes 1's complement in D0
        MOVE.W  D0,sum      ; store

4. NEG gives us 2's complement

        ...
        MOVE.W  num,D0      
        NEG.W   D0
        SUB.W   #1,D0
        MOVE.W  D0,num
        ...

Addressing Modes

  • Defines how data is accessed
  • Orthogonality property: every addressing mode would work with every instruction
  • Effective address, <ea> or EA; indicates how the address of the operand is calculated
  • All addressing modes, except immediate have an EA
ADD   Dn,<ea>      ; OR
ADD   <ea>,Dn

Basic Addressing Modes

1. Immediate

  • # indicates that the item that follows is immediate data (Ex. #4, #The, #%1001)
  • <ea> = none
MOVE.B    #25,D0     ; where D0 is 00 00 00 19
                     ; #25 = $19
 
ADDI.B    #1,D2

2. Register Direct (Dn or An)

  • Operand is a general register
  • <ea> = Rn
MOVE.B    #25,D0      ; direct
 
ADDI.B    #1,D2       ; immediate

3. Memory Direct

  • Operand is a variable +- constant expression (lazily evaluated)
  • The address of the operand is part of the instruction
  • <ea> = address
        ...
        MOVE.B  D0,num1     ; memory direct
 
num1    DC.B    $05

Also:

MOVE.B    25,D0       ; will move whatever is in memory location 25 to D0
  • (xxx).W or Abs.W (stored as 16 bits) [absolute short]
    • Ranges from $0 to 7FFF and $8000 to $FFFF
  • (xxx).L or Abs.L (stored as 32 bits) [absolute long]
  • Abs. short generates the more compact machine code

Example Shift a 16-variable left one bit in place

  • Input: num(W) = $57B6 = 0101 0111 1011 0110

  • Output: num(W) = $AF6C = 1010 1111 0110 1100

  • Algorithm:

    • D0 <- num
    • D0 <- leftshift(D0)
    • num <- D0

1.

          MOVE.W  num,D0      ; get value
          LSL.W   #1,D0       ; shift by 1
          MOVE.W  D0,num      ; and store
 
          STOP    #$2700
 
num       DC.W    $57B6
 
          END     START

2.

          ...
          LSL.W   num
          ...

3.

          ...
          MOVE.W  num,D0
          MOVE.W  #1,D1
          LSL.W   D1,D0
          MOVE.W  D0,num

4. Not ideal since multiplication is expensive

          ...
          MULU.W  #2,D0

5. Typically, use a shift

          ...
